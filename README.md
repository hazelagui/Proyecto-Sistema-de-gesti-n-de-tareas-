# Proyecto-Sistema-de-gesti-n-de-tareas-
#Requisitos del proyecto 
1. El sistema debe permitir registrar proyectos, tareas, responsables, cronograma, riesgos, costos, entre otros.
2. Los usuarios deben registrarse y acceder al sistema mediante una pantalla de inicio de sesión.
3. Los administradores pueden asignar tareas a los miembros del equipo.
4. Los usuarios deben poder actualizar el estado de sus tareas y añadir comentarios.
5. El sistema debe evaluar los costos según atrasos o avances en los proyectos.
6. El sistema debe clasificar los riesgos por color según su probabilidad e impacto (Verde=bajo, Amarillo=Medio,
Rojo=Alto).
7. El sistema debe manejar múltiples conexiones concurrentes.
8. El sistema debe enviar notificaciones según el avance de los hitos más importantes del proyecto a los
interesados.
9. La aplicación debe mostrar diferentes vistas según el rol del usuario conectado (administrador o colaborador).
10. Los administradores pueden generar reportes sobre el progreso de los proyectos.
11. Los usuarios deben poder ver un tablero con las tareas asignadas y sus fechas de vencimiento.
12. El sistema debe enviar recordatorios automáticos para tareas próximas a vencerse.
